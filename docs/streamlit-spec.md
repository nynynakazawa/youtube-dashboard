# Streamlit ダッシュボード仕様（初版）

## 目的
- ざわ研究チーム内でチャンネル/動画の集計を素早く検証するための分析ビュー。
- 初期段階ではざわ本人が利用、将来的には他研究者にも共有可能な設計とする。

## ページ構成
1. **概要タブ**
   - チャンネル概要メトリクス（登録者数/総再生数/動画数/直近更新日）
   - チャンネル別比較テーブル（複数チャンネルを並べて比較）
2. **曜日×時間帯ヒートマップ**
   - 軸: 曜日（縦）× 公開時間帯（横）
   - 値: 平均再生数 or エンゲージメント率
3. **動画特性分析**
   - 散布図: 動画長さ（秒） vs 再生数（バブルサイズ = 高評価数）
   - 30日成長カーブを重ねて比較できるラインチャート
4. **タグ別パフォーマンス**
   - タグ/キーワードごとの平均再生数・CTR・コメント数
   - 上位タグをランキング形式で表示

## データ取得
- すべて RDS（MySQL/PostgreSQL）から直接取得。
- 代表的な SQL 例（自然言語）:
  - 「指定チャンネルの動画を投稿日とカテゴリで集計し、平均再生数を算出」
  - 「動画 ID ごとに 30 日間の view 累積を日次で集約」
  - 「タグテーブルと動画テーブルを JOIN してタグ別の KPI を出す」

## UI 方針
- Streamlit のタブ/サイドバーを活用し、期間フィルタ・チャンネル選択・指標切替を配置。
- グラフは Plotly or Altair を利用（インタラクティブ前提）。
- 重要指標はカード表示で上部に常時固定。
